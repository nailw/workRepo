# 24039/24040 核料规则开发记录与最终源码

> 整理范围：2026-07-15 至 2026-07-17  
> 任务：24039“激素/高活物料车间隔离规则”、24040“加工贸易手册号规则-库存中的物料库位选择”

## 1. 最终业务结论

### 24039：产品禁用库存库位

在 KTProductRouting 中维护产品禁用库位，核料时只过滤库存类供应：

- MaterialDemand.productId 决定当前产品的禁用库位；
- MaterialSupply.sapStockingPointId 保存ERP库存库位；
- 普通库存、待检库存和半成品库存都必须复制库存库位；
- 采购订单、采购申请、采购计划、生产计划和本车间产出不受该规则影响；
- KT 的 NoLocationId 为空时，不限制库存；
- 多个禁用库位需要拆分、去空格和去重；
- 普通核料和冻结分配重占都必须校验24039。

### 24040：加工贸易手册号和库存库位

仅用于国际订单成品+BOM半成品场景：

- 按半成品代码查询 KTProcessingTradeHandBook；
- 按 createDate 升序选择优先级最高的手册；
- 当前手册对应的物料代码和库位还有可用库存时，只保留这组库存；
- 当前手册库存用完后，下一批核料选择下一本手册；
- 如果最早手册库位被24039禁用，先排除该库位，再选择下一本手册；
- 采购订单、采购申请、生产计划等非库存供应不受24040过滤；
- 冻结重占也必须校验24040，不能让旧冻结结果绕过手册号规则。

## 3. 已确认的DO语言规范

1. 排序使用 .sortedBy{}，不是 .sortBy{}。
2. 普通 if 可以写多个条件。
3. firstOrNull{} 内不要定义多步临时变量并执行复杂逻辑。
4. 复杂查找使用 forEach、标记变量和普通 if。
5. 每个 if 都必须使用大括号。
6. 代码减少无意义换行。
7. 不使用 !list.filter{...}.isEmpty() 这种复杂链式判断。
8. KT对象不直接使用 new KTProcessingTradeHandBook() 作为空占位对象，使用字符串字段和布尔变量保存选择结果。

## 4. 查询手册号

~~~java
static List<KTProcessingTradeHandBook> queryProcessingTradeHandBooks(Context context, String semiFinishedId) {
    var result = new List<KTProcessingTradeHandBook>()
    var targetSemiFinishedId = semiFinishedId ?? ""

    if (targetSemiFinishedId == "") {
        return result
    }

    result = context.KT<KTProcessingTradeHandBook>().filter{
        (it.semiFinishedId ?? "") == targetSemiFinishedId &&
        (it.manualNumber ?? "") != ""
    }.sortedBy{ it.createDate }

    return result
}
~~~

## 5. 24040手册号库存过滤

~~~java
static void filterProcessingTradeHandBookSuppliesInPlace(List<MaterialSupply> supplies, List<KTProcessingTradeHandBook> handBooks) {
    if (handBooks.isEmpty()) {
        return
    }

    var hasSelectedHandBook = false
    var selectedMaterialId = ""
    var selectedLocationId = ""

    handBooks.forEach{ handBook ->
        if (!hasSelectedHandBook) {
            var targetMaterialId = handBook.materialId ?? ""
            var targetLocationId = handBook.warehouseLocation ?? ""
            var hasAvailableSupply = false

            if (targetMaterialId != "" && targetLocationId != "") {
                supplies.forEach{ supply ->
                    if (!hasAvailableSupply &&
                        MaterialSupply.isInventorySupply(supply) &&
                        (supply.productId ?? "") == targetMaterialId &&
                        (supply.sapStockingPointId ?? "") == targetLocationId &&
                        supply.leftQuantity > Constant.EPS) {
                        hasAvailableSupply = true
                    }
                }
            }

            if (hasAvailableSupply) {
                selectedMaterialId = targetMaterialId
                selectedLocationId = targetLocationId
                hasSelectedHandBook = true
            }
        }
    }

    if (!hasSelectedHandBook) {
        supplies.removeIf{ MaterialSupply.isInventorySupply(it) }
        return
    }

    supplies.removeIf{ supply ->
        MaterialSupply.isInventorySupply(supply) &&
        ((supply.productId ?? "") != selectedMaterialId ||
         (supply.sapStockingPointId ?? "") != selectedLocationId)
    }
}
~~~

## 6. 公共供应过滤最终实现

24040候选物料必须在初始化时从全部 inputSupplies 中取得，不能先按 materialDemand.materialId 分组后再过滤，否则会丢失其他手册对应的物料代码。

24040过滤必须放在24039、数量、生产月份、有效期、供应商和上游车间等条件之后，使用最终可用供应进行手册选择。

~~~java
var supplyMap = inputSupplies.groupBy{ it.productId }
var isGuojiChengpingUseBomBanchengping = MaterialSupply.isGuojiChengpingUseBomBanchengping(materialDemand, conetxt)
var handBooks = new List<KTProcessingTradeHandBook>()
var outputSupplies = new List<MaterialSupply>()

if (isGuojiChengpingUseBomBanchengping) {
    handBooks = MaterialSupply.queryProcessingTradeHandBooks(conetxt, materialDemand.materialId)
    var candidateMaterialIds = handBooks.map{ it.materialId ?? "" }.filter{ it != "" }
    var demandMaterialId = materialDemand.materialId ?? ""

    if (!candidateMaterialIds.contain(demandMaterialId)) {
        candidateMaterialIds.add(demandMaterialId)
    }

    outputSupplies = inputSupplies.filter{
        candidateMaterialIds.contain(it.productId ?? "")
    }
} else {
    outputSupplies = supplyMap.getOrDefault(materialDemand.materialId, new List<MaterialSupply>())
}

var noLocationIds = MaterialSupply.queryNoLocationIdsByProductId(
    conetxt,
    materialDemand.productId,
    materialDemand.pieceSizeQuantity,
    materialDemand.routingId
)

var availableSupplies = new List<MaterialSupply>()

outputSupplies.forEach{ supply ->
    var locationId = supply.sapStockingPointId ?? ""
    var isForbidden = MaterialSupply.isInventorySupply(supply) &&
        locationId != "" &&
        noLocationIds.contain(locationId)

    if (!isForbidden) {
        availableSupplies.add(supply)
    }
}

outputSupplies = availableSupplies

var productDemand = conetxt.KT<KTSingleResourceSpeedExtend>().filter{
    it.productId == materialDemand.productId
}

if (!productDemand.isEmpty()) {
    var scheduledPieceSteps = conetxt.Group<PieceStep>().filter{
        it.productId == materialDemand.productId &&
        it.pieceId == materialDemand.pieceId &&
        it.isScheduled == true
    }

    if (!scheduledPieceSteps.isEmpty()) {
        scheduledPieceSteps.forEach{ p ->
            var usedResourceGroupId = conetxt.Group<Resource>().filter{
                it.resourceId == p.usedResourceId
            }.firstOrNull()?.resourceGroupId ?? ""

            if (usedResourceGroupId != "") {
                var materialsID = productDemand.filter{
                    it.resourceGroupId == usedResourceGroupId
                }.firstOrNull()?.NoMaterialsID ?? ""

                if (materialsID != "") {
                    outputSupplies.removeIf{ it.productId == materialsID }
                }
            }
        }
    }
}

var banchengpingIds = conetxt.KT<KTProductRouting>().filter{
    (it?.productType1 ?? "") == "B" &&
    it.customType == "国内订单"
}.map{ it.productId }

var isDomesticSemiDemand = banchengpingIds.contain(materialDemand.materialId)

if (isGuojiChengpingUseBomBanchengping) {
    outputSupplies.removeIf{
        MaterialSupply.getGuojiBanchengpingAvailableQty(it) <= Constant.EPS
    }
} else if (isDomesticSemiDemand) {
    outputSupplies.removeIf{
        it.leftQuantityVritual <= Constant.EPS
    }
} else {
    outputSupplies.removeIf{
        it.leftQuantity < leastQty
    }
}

if (isGuojiChengpingUseBomBanchengping) {
    MaterialSupply.filterGuojiBanchengpingByProductionMonthInPlace(
        outputSupplies,
        materialDemand,
        conetxt
    )
}

if (materialDemand.forbiddenMix &&
    !isGuojiChengpingUseBomBanchengping) {
    if (isDomesticSemiDemand) {
        outputSupplies.removeIf{
            it.leftQuantityVritual <= Constant.EPS
        }
    } else {
        outputSupplies.removeIf{
            it.leftQuantity < materialDemand.demandQuantity
        }
    }
}

if (materialDemand.expireConstraint) {
    outputSupplies.removeIf{
        MaterialSupply.isInventorySupply(it) &&
        it.expireDate < materialDemand.dueDate
    }
}

if (materialDemand.supplierConstraint) {
    var supplierIds = Utils.splitStringToStringList(materialDemand.supplierIds)
    outputSupplies.removeIf{
        !supplierIds.contain(it.supplierId)
    }
}

if (materialDemand.upStreamWorkshopConstraint) {
    var workshopIds = Utils.splitStringToStringList(materialDemand.upStreamWorkshopIds)
    outputSupplies.removeIf{
        !workshopIds.contain(it.workShopId)
    }
}

if (isGuojiChengpingUseBomBanchengping) {
    MaterialSupply.filterProcessingTradeHandBookSuppliesInPlace(
        outputSupplies,
        handBooks
    )
}

return outputSupplies
~~~

## 7. 冻结分配重占最终实现

冻结重占不经过公共供应查询，所以必须单独校验24039和24040。

~~~java
var dh = context().getDbHelper()
var needUpdateDemands = new List<MaterialDemand>()
var needUpdateSupplies = new List<MaterialSupply>()
var invalidFrozenFulfills = new List<MaterialFulfill>()

var qfulfill = QMaterialFulfill.qmaterialfulfill
var frozenFulfills = dh.dslSelectFrom(qfulfill).where(qfulfill.frozenFlag.eq(true)).findAll()
var supplies = MaterialSupply.queryAllData(context())
var supplyMap = supplies.groupBy{ it.uniqueId }

frozenFulfills.groupBy{ it.demandUniqueId }.forEach((demandUniqueId, fulfills) -> {
    var demand = MaterialDemand.findByUniqueId(context(), demandUniqueId)

    if (demand == null) {
        invalidFrozenFulfills.addAll(fulfills)
        return
    }

    var firstPriorityDemand = MaterialDemand.findFirstReplacePriorityByMaterialDemand(context(), demand)
    var noLocationIds = MaterialSupply.queryNoLocationIdsByProductId(
        context(),
        demand.productId,
        demand.pieceSizeQuantity,
        demand.routingId
    )

    var isGuojiChengpingUseBomBanchengping =
        MaterialSupply.isGuojiChengpingUseBomBanchengping(
            demand,
            context()
        )

    var handBooks = new List<KTProcessingTradeHandBook>()
    var candidateMaterialIds = new List<String>()

    if (isGuojiChengpingUseBomBanchengping) {
        handBooks = MaterialSupply.queryProcessingTradeHandBooks(
            context(),
            demand.materialId
        )

        candidateMaterialIds = handBooks.map{
            it.materialId ?? ""
        }.filter{
            it != ""
        }

        var demandMaterialId = demand.materialId ?? ""

        if (!candidateMaterialIds.contain(demandMaterialId)) {
            candidateMaterialIds.add(demandMaterialId)
        }
    }

    var hasValidFulfill = false

    fulfills.forEach{ fulfill ->
        var supply = supplyMap.getOrDefault(
            fulfill.supplyUniqueId,
            new List<MaterialSupply>()
        ).firstOrNull()

        if (supply == null) {
            invalidFrozenFulfills.add(fulfill)
            return
        }

        var locationId = supply.sapStockingPointId ?? ""
        var invalidLocation = MaterialSupply.isInventorySupply(supply) &&
            locationId != "" &&
            noLocationIds.contain(locationId)

        if (invalidLocation) {
            invalidFrozenFulfills.add(fulfill)
            return
        }

        if (isGuojiChengpingUseBomBanchengping &&
            !handBooks.isEmpty() &&
            MaterialSupply.isInventorySupply(supply)) {
            var handBookSupplies = supplies.filter{
                candidateMaterialIds.contain(it.productId ?? "")
            }

            var legalHandBookSupplies = new List<MaterialSupply>()

            handBookSupplies.forEach{ handBookSupply ->
                var handBookLocationId = handBookSupply.sapStockingPointId ?? ""
                var handBookInvalidLocation =
                    MaterialSupply.isInventorySupply(handBookSupply) &&
                    handBookLocationId != "" &&
                    noLocationIds.contain(handBookLocationId)

                if (!handBookInvalidLocation) {
                    legalHandBookSupplies.add(handBookSupply)
                }
            }

            MaterialSupply.filterProcessingTradeHandBookSuppliesInPlace(
                legalHandBookSupplies,
                handBooks
            )

            if (!legalHandBookSupplies.contain(supply)) {
                invalidFrozenFulfills.add(fulfill)
                return
            }
        }

        demand.fulfillQuantity += fulfill.quantity
        hasValidFulfill = true

        if (firstPriorityDemand != null) {
            var demandQty = demand.pieceStepQuantity * demand.inputFactor
            var ratio = demandQty > 0.0 ? fulfill.quantity / demandQty : 0.0
            firstPriorityDemand.leftRatio -= ratio
        }

        supply.leftQuantity -= fulfill.quantity
        supply.leftQuantity.roundHalfDown(7)

        if (!needUpdateSupplies.contain(supply)) {
            needUpdateSupplies.add(supply)
        }

        fulfill.availableDate = supply.availableReleaseDate
        fulfill.availableTime = supply.availableReleaseTime
    }

    if (hasValidFulfill) {
        if (!needUpdateDemands.contain(demand)) {
            needUpdateDemands.add(demand)
        }

        if (firstPriorityDemand != null &&
            firstPriorityDemand != demand &&
            !needUpdateDemands.contain(firstPriorityDemand)) {
            needUpdateDemands.add(firstPriorityDemand)
        }
    }
})

if (!invalidFrozenFulfills.isEmpty()) {
    dh.deleteBatch(invalidFrozenFulfills)
}

var validFrozenFulfills = frozenFulfills.filter{
    !invalidFrozenFulfills.contain(it)
}

if (!needUpdateDemands.isEmpty()) {
    dh.updateBatch(needUpdateDemands)
}

if (!needUpdateSupplies.isEmpty()) {
    dh.updateBatch(needUpdateSupplies)
}

if (!validFrozenFulfills.isEmpty()) {
    dh.updateBatch(validFrozenFulfills)
}
~~~

## 8. 测试方案

测试前确认 MaterialSupply.sapStockingPointId 已正确生成。每个场景测试前恢复供应数量，避免上一个场景扣减的 leftQuantity 影响后续场景。

### 普通核料

| 场景 | 数据 | 预期 |
|---|---|---|
| 最早手册有库存 | H001→物料10010016→6027 | 只保留6027 |
| 第一手册用完 | 6027剩余量为0 | 切换H002 |
| 第一手册库位被24039禁用 | NoLocationId=6027 | 跳过H001，选择H002 |
| KT无手册数据 | 对应半成品无记录 | 不执行24040 |
| 采购/生产计划供应 | 非库存供应 | 不受24040影响 |
| 多个禁用库位 | NoLocationId=6027,6029 | 两个库位均过滤 |

### 冻结重占

| 场景 | 预期 |
|---|---|
| 冻结供应不存在 | 不增加fulfillQuantity，删除冻结记录 |
| 冻结库存命中24039 | 不扣减库存，删除冻结记录 |
| 冻结库存不符合24040手册 | 不增加满足量，删除冻结记录 |
| 冻结供应合法 | 增加需求满足量，扣减供应剩余量，更新时间 |
| 冻结供应为采购/生产计划 | 不受库存库位规则影响 |

重点核对：

MaterialSupply.leftQuantity
MaterialSupply.sapStockingPointId
MaterialFulfill.supplyUniqueId
MaterialFulfill.frozenFlag

## 9. 修正记录

1. 初始24039方案只在普通核料入口过滤，后补充冻结重占校验。
2. demand.fulfillQuantity += fulfill 从供应存在性校验之前移动到校验之后。
3. 无效冻结分配不再继续保留，统一删除。
4. 24040不能在按需求物料分组后再过滤，必须先把手册对应物料加入候选供应。
5. 24040过滤从普通供应过滤前移动到所有最终可用条件之后。
6. 冻结24040校验中先排除24039禁用库位，避免优先选择已被禁用的早期手册。
7. 手册查询从每条冻结分配查询一次改为每个需求查询一次。
8. 已确认DO语法：使用 .sortedBy{}；firstOrNull{}不写多步复杂闭包；普通 if 允许多个条件；所有 if 必须有大括号。
9. 调试日志只作为临时排查手段，验证完成后移除，不保留业务日志。

## 10. 完成标准

- 库存供应正确带出ERP库位；
- 普通核料正确过滤24039禁用库位；
- 国际BOM半成品按手册号、创建时间、物料代码和库位选择；
- 第一手册库存用完后能切换下一手册；
- 24039禁用第一本手册时能继续选择合法手册；

- 供应不存在或规则非法时不增加需求满足量；
- 非库存供应不受库存库位规则影响；
- 测试场景全部通过后再移除临时调试数据。
