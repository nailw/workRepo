# 24039 激素/高活物料车间隔离规则开发方案

## 1. 目标

在产品路径 `KTProductRouting` 中维护产品禁用库存库位，核料时只对库存类供应动态过滤；采购订单、采购申请、采购计划、其他场景生产计划和本车间产出不受影响。

示例：产品 `12020228` 禁用库位 `3411`，该产品只能使用 `3410/3412`；产品 `12020071` 未配置禁用库位，核料逻辑保持原样。

核心原则：不能在库存同步阶段全局删除库位，必须按 `MaterialDemand.productId` 在供需匹配时过滤。

## 2. 数据模型

### KTProductRouting

新增字段：

```text
NoLocationId: String，可为空；多个值使用逗号分隔，例如 3411,3413
```

### MaterialSupply

新增字段：

```text
sapStockingPointId: String，保存库存供应对应的 ERP 库位代码
```

`InventoryIn.sapStockingPointId` 已有字段，不需要新增。

## 3. 库存转换

在以下三个方法中统一复制库位：

```text
updateAttributeByInventoryAvailableNode
updateAttributeByInventoryDetectNode
updateAttributeBySemiInventoryNode
```

三个方法的库存基础字段后增加：

```java
obj.sapStockingPointId = inventory.sapStockingPointId ?? ""
```

必须覆盖可用库存、待检库存和半成品库存。

## 4. 产品禁用库位查询

在 `MaterialSupply` 新增：

```java
static queryNoLocationIdsByProductId(context, productId) {
    var result = new List<String>()
    if ((productId ?? "") == "") return result
    context.KT<KTProductRouting>().filter { it != null && it.productId == productId }.forEach { routing ->
        Utils.splitStringToStringList(routing.NoLocationId ?? "").forEach { locationId ->
            var id = locationId ?? ""
            if (id != "" && !result.contain(id)) result.add(id)
        }
    }
    return result
}
```

该方法合并同一产品多条路径记录中的禁用库位并去重；空值、空字符串不参与过滤。

## 5. 公共供应过滤

在 `queryAvailableFromInputSupplies()` 中，按物料取得候选供应后，立即增加：

```java
var supplyMap = inputSupplies.groupBy { it.productId }
var outputSupplies = supplyMap.getOrDefault(materialDemand.materialId, new List<MaterialSupply>())
var noLocationIds = MaterialSupply.queryNoLocationIdsByProductId(conetxt, materialDemand.productId)
if (!noLocationIds.isEmpty()) {
    outputSupplies.removeIf { supply ->
        MaterialSupply.isInventorySupply(supply) &&
        (supply.sapStockingPointId ?? "") != "" &&
        noLocationIds.contain(supply.sapStockingPointId)
    }
}
```

后面继续执行原有规则：设备组禁用物料、国际/国内半成品数量、生产月份、不可混批、有效期、供应商和上游车间。

过滤顺序：

```text
按 materialId 取候选供应
→ 24039 禁用库位过滤
→ 设备组过滤
→ 数量/生产月份过滤
→ 不可混批
→ 有效期、供应商、上游车间
```

`isInventorySupply()` 保持：

```java
return supply.dataSource == ModelConstants.FromInventoryAvailable ||
       supply.dataSource == ModelConstants.FromInventoryDetect
```

因此采购和生产计划等无实际库存库位的供应不会被误删。普通物料、替代料和配套组均经过该公共入口，统一生效。

## 6. 冻结分配重占

`reOccupyFrozenMaterialFulfill()` 不经过公共供应查询，必须单独校验。新增：

```java
var invalidFrozenFulfills = new List<MaterialFulfill>()
```

在更新需求之前先查供应、校验库位：

```java
fulfills.forEach { fulfill ->
    var supply = supplyMap.getOrDefault(fulfill.supplyUniqueId, new List<MaterialSupply>()).firstOrNull()
    if (supply == null) {
        invalidFrozenFulfills.add(fulfill)
        return
    }
    var noLocationIds = MaterialSupply.queryNoLocationIdsByProductId(context(), demand.productId)
    var invalidLocation = MaterialSupply.isInventorySupply(supply) &&
        (supply.sapStockingPointId ?? "") != "" &&
        noLocationIds.contain(supply.sapStockingPointId)
    if (invalidLocation) {
        invalidFrozenFulfills.add(fulfill)
        return
    }

    demand.fulfillQuantity += fulfill.quantity
    if (firstPriorityDemand != null) {
        var demandQty = demand.pieceStepQuantity * demand.inputFactor
        firstPriorityDemand.leftRatio -= demandQty > 0.0 ? fulfill.quantity / demandQty : 0.0
    }
    supply.leftQuantity -= fulfill.quantity
    supply.leftQuantity.roundHalfDown(7)
    if (!needUpdateSupplies.contain(supply)) needUpdateSupplies.add(supply)
    fulfill.availableDate = supply.availableReleaseDate
    fulfill.availableTime = supply.availableReleaseTime
}
```

非法冻结结果包括：

- 冻结供应已不存在；
- 库位命中产品禁用库位。

处理完成后先删除非法冻结结果，再更新合法结果：

```java
if (!invalidFrozenFulfills.isEmpty()) dh.deleteBatch(invalidFrozenFulfills)
var validFrozenFulfills = frozenFulfills.filter { !invalidFrozenFulfills.contain(it) }
dh.updateBatch(needUpdateDemands)
dh.updateBatch(needUpdateSupplies)
dh.updateBatch(validFrozenFulfills)
```

如果项目的 `DbHelper` 没有 `deleteBatch`，使用现有 `MaterialFulfill` 批量删除方法替换，原则不变。

## 7. 核料流程影响

```text
同步供应/需求
→ 删除未冻结分配
→ 校验并重占合法冻结分配
→ 普通、替代料、配套组分配
→ 计算最早可用时间
→ 发送 DM
```

24039 只改变库存供应候选，不改变供应数量计算、供应排序、替代组优先级和齐套时间计算方式。

## 8. 自测用例

| 编号 | 场景 | 预期 |
|---|---|---|
| 1 | 未配置 NoLocationId | 与改造前一致 |
| 2 | 禁用 3411 | 3411 过滤，3410/3412 可用 |
| 3 | 禁用 3411,3413 | 两个库位均过滤 |
| 4 | 同一库位被不同产品使用 | 只影响配置禁用的产品 |
| 5 | 可用、待检、半成品库存 | 三类库存统一按库位过滤 |
| 6 | 采购订单/采购申请/生产计划 | 不受库位规则影响 |
| 7 | 替代料 | 替代库存同样过滤 |
| 8 | 配套组 | 临时满足率和正式分配结果一致 |
| 9 | 冻结分配命中禁用库位 | 删除冻结结果并重新核料 |
| 10 | 冻结供应已不存在 | 不增加需求满足量，重新核料 |
