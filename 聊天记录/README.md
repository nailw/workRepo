# 聊天记录归档

本目录按日期保存与项目相关的聊天记录、关键决策和执行结果。

## 文件命名

使用 `YYYY-MM-DD.md`。同一天的多轮对话继续追加到同一文件，并按时间或主题分节。

## 记录原则

- 优先保存用户要求的原文；
- 助手回复无法完整恢复时，只写处理摘要并明确标注；
- 每次记录必须列出已修改文件、结果和待办事项；
- 涉及敏感信息时，遵守 [备份与归档规范](../docs/备份与归档规范.md) 的脱敏要求；
- 禁止把密钥、密码、令牌和未脱敏机密写入公开仓库。

> 注意：当前仓库为公开仓库。完整敏感聊天应保存到私有仓库。
