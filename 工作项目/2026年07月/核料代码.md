## 2. 已提供的代码

### 2.1 客户端入口 `checkInputMaterialForClient`

> **代码说明**：核料客户端入口，按当前 Context 获取 MaterialManager 单例并启动核料。
>
> **关键条件/限制**：要求 Context 可用；自身不做业务判断。
>
> **规则对应**：对应 3.2 的入口和完整八阶段流程。

```java
var materialManager = MaterialManager.getInstance(context)
materialManager.checkInputMaterial()
```

### 2.2 核料主流程 `checkInputMaterial`

> **代码说明**：核料总编排，依次完成供需同步、冻结处理、分配、工序回写和 DM 同步。
>
> **关键条件/限制**：核料开关关闭时立即退出；各阶段记录耗时。
>
> **规则对应**：对应 3.2 全流程及 3.3.1 场景开关。

```java
var time1 = DateTime.now()
//0.0判断是否开启了核料开关，否则不会走以下逻辑及生成相关实例
if (!this.enabledCheckInputMaterial()) {
    return
}
//1.0同步供应和需求---35s->17
this.synDemandAndSupply()
var time2 = DateTime.now()
Utils.logRunningDuration("同步供应和需求",time1,time2)
//2.0删除现有的未冻结数据，并更新冻结占用---1s
this.clearUnfrozenAndReOccupyFrozenMaterialFulfills()
var time3 = DateTime.now()
Utils.logRunningDuration("删除现有的未冻结数据，并更新冻结占用",time2,time3)
//3.0物料分配---361s->87
var pieceSteps = Group<PieceStep>().toList()
this.assignSuppliesToMaterialDemands()
var time4 = DateTime.now()
Utils.logRunningDuration("物料分配",time3,time4)
//4.0根据分配，计算物料最早可用时间---1392s
this.updateFulfillTimeToPieceStepTime(pieceSteps)
var time5 = DateTime.now()
Utils.logRunningDuration("根据分配，计算物料最早可用时间",time4,time5)
//5.0物料分配发送给DM---15s
this.sendFulfillsToDM()
var time6 = DateTime.now()
Utils.logRunningDuration("0物料分配发送给DM",time5,time6)
```

### 2.6 工序BOM模式 `synInputAndOutOnStepByOperationBOM`

> **代码说明**：按工序 BOM 生成投入，并在末工序生成产出。
>
> **关键条件/限制**：递归展开 BOM；过期旧投入/产出通过 isDirty 清理。
>
> **规则对应**：对应 3.3.3 工序 BOM 规则。

```java
if (!this.enabledCheckInputMaterial()) {
    return
}
var data = (List<Map<String, Any>>) Utils.getDataFromDM(context(),"DataSenderManager","sendI23_ImportBomAndOperationToPP",null)
var bomIndex = data.groupBy{ Utils.anyToString(it["queryWorkOrderId"]) }
Group<InputOnStep>().forEach{ it.isDirty = true }
Group<OutputOnStep>().forEach{ it.isDirty = true }
var toAddOutput = new List<OutputOnStep>()
Group<WorkOrder>().forEach{ workOrder ->
    var operationBoms = bomIndex.getOrDefault(workOrder.queryOperationBomId,new List<Map>())
    operationBoms.forEach{ operationBom ->
        this.createInputByOperationBOM(workOrder,operationBoms,operationBom,1.0)
    }
    var lastWorkOrderStep = workOrder.lastWorkOrderStep
    if (lastWorkOrderStep != null) {
        var output = indexing(OutputOnStep.indexByOwnerAndProductId,lastWorkOrderStep,workOrder.productId).firstOrNull()
        if (output == null) {
            output = new OutputOnStep(lastWorkOrderStep)
            toAddOutput.add(output)
        }
        output.isDirty = false
        if (operationBoms.isNotEmpty()) {
            output.updateAttributeByOperationBom(operationBoms.firstOrNull())
        }
    }
}
Group<InputOnStep>().removeIf{ it.isDirty }
Group<OutputOnStep>().removeIf{ it.isDirty }
Group<OutputOnStep>().addAll(toAddOutput)
```

### 2.7 普通BOM模式 `synInputAndOutOnStepByBOM`

> **代码说明**：按普通 BOM、KT 特殊约束和订单路径生成投入/产出。
>
> **关键条件/限制**：特殊物料放指定步骤，普通物料放首步骤，产出放末步骤。
>
> **规则对应**：对应 3.3.3 普通 BOM 规则。

```java
if (!this.enabledCheckInputMaterial()) {
    return
}
var semiProductIds = BillOfMaterial.querySemiProductIdsByBom(context())
Group<InputOnStep>().forEach{ it.isDirty = true }
Group<OutputOnStep>().forEach{ it.isDirty = true }
var toAddInput = new List<InputOnStep>()
var toAddOutput = new List<OutputOnStep>()
Group<WorkOrder>().groupBy{ it.productId + it.bomVer }.forEach((productIdAbomver,orders) -> {
    var firstOrder = orders.firstOrNull()
    if (firstOrder == null) {
        return
    }
    var productId = firstOrder.productId
    var ktRows = KT<KTInputConstraint>().select(it.productId == productId,it.stepSequenceNr != "")
    //1.0对有特殊约束的KT，在对应步骤创建input
    ktRows.forEach{ ktRow ->
        var boms = BillOfMaterial.queryByProductIdAndRawMaterialId(context(),productId,ktRow.rawMaterialId).filter{ it.isEffective }
        if (boms.isEmpty()) {
            return
        }
        indexing(WorkOrderStep.indexByProductIdAndSeqNr,productId,ktRow.stepSequenceNr).forEach{ workOrderStep ->
            if (workOrderStep == null) {
                return
            }
            var order = workOrderStep.owner
            if (order == null || !this.canUseBomByInputProductId(order,semiProductIds,ktRow.rawMaterialId)) {
                return
            }
            boms.forEach{ bom ->
                var input = indexing(InputOnStep.indexByOwnerAndMaterialIdAndLineNr,workOrderStep,bom.rawMaterialId,bom.lineNr).firstOrNull()
                if (input == null) {
                    input = new InputOnStep(workOrderStep)
                    toAddInput.add(input)
                }
                input.isDirty = false
                input.updateAttributeByBom(Utils.dataValuesToMap(bom))
            }
        }
    }
    //2.0根据产品代码查找BOM，并对普通BOM在第一步创建input
    var bomsData = BillOfMaterial.queryByProductId(context(),productId).filter{ it.isEffective }
    var hasConstraintRawMaterialIds = ktRows.map{ it.rawMaterialId }
    var commonBoms = bomsData.filter{ !hasConstraintRawMaterialIds.contain(it.rawMaterialId) }
    orders.forEach{ order ->
        indexing(Piece.indexByOwner,order).forEach{ piece ->
            var workOrderStep = piece.firstWorkOrderStep
            if (workOrderStep == null) {
                return
            }
            commonBoms.forEach{ bom ->
                if (!this.canUseBomByInputProductId(order,semiProductIds,bom.rawMaterialId)) {
                    return
                }
                var input = indexing(InputOnStep.indexByOwnerAndMaterialIdAndLineNr,workOrderStep,bom.rawMaterialId,bom.lineNr).firstOrNull()
                if (input == null) {
                    input = toAddInput.find{ it.owner == workOrderStep && it.materialId == bom.rawMaterialId && it.lineNr == bom.lineNr }
                }
                if (input == null) {
                    input = new InputOnStep(workOrderStep)
                    toAddInput.add(input)
                }
                input.isDirty = false
                input.updateAttributeByBom(Utils.dataValuesToMap(bom))
            }
        }
    }
    //3.0对于所有订单的所有末端步骤，创建output
    orders.forEach{ order ->
        var orderProductId = order.productId
        indexing(Piece.indexByOwner,order).forEach{ piece ->
            var workOrderStep = piece.selectedWorkOrderSteps.sortedBy{ it.sequenceNr }.lastOrNull()
            if (workOrderStep == null) {
                return
            }
            var output = indexing(OutputOnStep.indexByOwnerAndProductId,workOrderStep,orderProductId).firstOrNull()
            if (output == null) {
                output = toAddOutput.find{ it.owner == workOrderStep && Utils.anyToString(it.productId) == Utils.anyToString(orderProductId) }
            }
            if (output == null) {
                output = new OutputOnStep(workOrderStep)
                toAddOutput.add(output)
            }
            output.isDirty = false
            if (bomsData.isNotEmpty()) {
                output.updateAttributeByBom(Utils.dataValuesToMap(bomsData.firstOrNull()))
            }
        }
    }
})
Group<InputOnStep>().removeIf{ it.isDirty }
Group<OutputOnStep>().removeIf{ it.isDirty }
Group<InputOnStep>().addAll(toAddInput)
Group<OutputOnStep>().addAll(toAddOutput)
```

### 2.10 库存转供应 `synInventoryToMaterialSupply`

> **代码说明**：把库存拆分为半成品库存、可用库存和待检库存供应。
>
> **关键条件/限制**：数量必须大于 EPS；待检库存释放时间叠加检验周期。
>
> **规则对应**：对应 3.3.4 供应来源与有效性。

```java
var inventories = InventoryIn.queryAllData(context())
var toAdd = new List<MaterialSupply>()
var dh = context().getDbHelper()
var semiProductIds = BillOfMaterial.querySemiProductIdsByBom(context())
inventories.groupBy{ it.productId }.forEach((productId,inventoryList) -> {
    var product = indexing(Product.indexByProductId,productId).firstOrNull()
    var detectDuration = product?.detectDuration ?? Duration.ZERO
    var canSelfProduction = product?.canSelfProduction ?? false
    var isBanChengPing = semiProductIds.contain(productId)
    inventoryList.forEach{ inventory ->
        if (isBanChengPing) {
            var semiQty = inventory.quantity
            if (semiQty > Constant.EPS) {
                var obj = new MaterialSupply()
                MaterialSupply.updateAttributeBySemiInventoryNode(obj,inventory)
                obj.detectDuration = detectDuration
                obj.canSelfProduction = canSelfProduction
                this.setSemiFinishedInventoryDate(obj,inventory)
                this.setSemiProductVirtualQuantity(obj,productId)
                MaterialSupply.updateSupplySortKey(obj)
                if (obj.supplyQuantity > Constant.EPS) {
                    toAdd.add(obj)
                }
            }
            return
        }
        var availableQty = inventory.quantity
        if (availableQty > Constant.EPS) {
            var obj = new MaterialSupply()
            MaterialSupply.updateAttributeByInventoryAvailableNode(obj,inventory)
            obj.detectDuration = detectDuration
            obj.canSelfProduction = canSelfProduction
            obj.availableReleaseTime = obj.availableTime
            obj.availableReleaseDate = obj.availableDate
            MaterialSupply.updateSupplySortKey(obj)
            if (obj.supplyQuantity > Constant.EPS) {
                toAdd.add(obj)
            }
        }
        var detectQty = inventory.detectQuantity
        if (detectQty > Constant.EPS) {
            var obj = new MaterialSupply()
            MaterialSupply.updateAttributeByInventoryDetectNode(obj,inventory)
            obj.detectDuration = detectDuration
            obj.canSelfProduction = canSelfProduction
            obj.availableReleaseTime = TimeCalc.timePlusDuration(obj.availableTime,detectDuration)
            obj.availableReleaseDate = obj.availableReleaseTime.toDate()
            MaterialSupply.updateSupplySortKey(obj)
            if (obj.supplyQuantity > Constant.EPS) {
                toAdd.add(obj)
            }
        }
    }
})
dh.insertBatch(toAdd)
```

### 2.15 扣除其他车间占用 `synMaterialSupplyAvailableQuantity`

> **代码说明**：扣除其他项目或车间已占用的供应数量。
>
> **关键条件/限制**：排除当前项目和场景；剩余量不小于0并保留7位精度。
>
> **规则对应**：对应 3.3.4 外部占用规则。

```java
if (!this.enabledCheckInputMaterial()) {
    return
}
var time1 = DateTime.now()
var data = (List<Map<String, Any>>) Utils.getDataFromDM(context(),"DataSenderManager","sendIPP_MaterialFulfillToPP",null)
var dh = context().getDbHelper()
var time2 = DateTime.now()
Utils.logRunningDuration("读取",time1,time2)
data.removeIf{ Utils.anyToString(it["ppProjectId"]) == context().projectCode() && Utils.anyToString(it["ppSceneId"]) == context().sceneCode() }
var time3 = DateTime.now()
Utils.logRunningDuration("移除",time2,time3)
var fulfillMap = data.groupBy{ Utils.anyToString(it["supplyUniqueId"]) }
var qSupply = QMaterialSupply.qmaterialsupply
var thisSceneId = context().sceneCode()
var supplies = dh.dslSelectFrom(qSupply).findAll()
var toUpdate = new List<MaterialSupply>()
supplies.forEach{ supply ->
    var fulfills = fulfillMap.getOrDefault(supply.uniqueId,new List<Map>())
    if (fulfills.isNotEmpty()) {
        toUpdate.add(supply)
        supply.occupiedByOtherWorkshop = fulfills.sumByDouble{ Utils.anyToDouble(it["quantity"]) }
        supply.availableQuantity = maxOf(0.0,supply.supplyQuantity - supply.occupiedByOtherWorkshop)
        supply.leftQuantity = supply.availableQuantity.roundHalfDown(7)
    }
}
var time4 = DateTime.now()
Utils.logRunningDuration("更新",time3,time4)
dh.updateBatch(supplies)
```

### 2.17 根据步骤投入生成需求 `synInputStepToMaterialDemand`

```java
        var supply = sortedSupplies.firstOrNull()
```

```java
        var supplyProductId = Utils.anyToString(supply.productId)

            : null

        if (currentDemand == null) continue

         * A库存 -> B库存 -> A生产/采购 -> B生产/采购。

        if (availableQty + Constant.EPS < remainQty) continue

        totalFulfillRatio += fulfillRatio

        if (!tempFlag) {

            fulfill.unFulfillQuantity = 0.0

            if (needSemiProductBatchPass) {

                supply.leftQuantityVritual = 0.0

                firstDemand.leftRatio -= fulfillRatio
                if (firstDemand.leftRatio < Constant.EPS) firstDemand.leftRatio = 0.0
            }
        }

        return (totalFulfillRatio, toAdd)

        splitPieceQtyList.add(batchQty)
        splitPieceMaterialUniqueIdList.add(Utils.anyToString(supply.uniqueId))
        commitIndex += 1
    }

    /*
     * 创建/更新 Piece，并保存每批实际使用的半成品 uniqueId。
     */
    if (!tempFlag && splitPieceQtyList.isNotEmpty()) {
        sourcePiece = context.Group<Piece>().find { it.pieceId == firstDemand.pieceId }
        if (sourcePiece == null || sourcePiece.owner == null) return tuple

        var workOrder = sourcePiece.owner
        var currentPieceCount = workOrder.pieces.size()

        if (currentPieceCount == guojiBatchCountLimit) {
            return (totalFulfillRatio, toAdd)
        }

        var maxSystemNum = workOrder.pieces.maxByOrNull {
            it.systemNum ?? 0
        }?.systemNum ?? 0

        var pieceIndex = 0
        splitPieceQtyList.forEach { pieceQty ->
            var calcPieceQty = pieceQty.roundHalfUp(6)
            var selectedMaterialUniqueId = splitPieceMaterialUniqueIdList[pieceIndex]

            if (calcPieceQty > Constant.EPS && !selectedMaterialUniqueId.isEmpty()) {
                if (pieceIndex == 0) {
                    sourcePiece.quantity = calcPieceQty
                    sourcePiece.selectedSemiProductUniqueId = selectedMaterialUniqueId
                } else {
                    maxSystemNum += 1
                    var newPiece = workOrder.createNewPiece(calcPieceQty, maxSystemNum)
                    newPiece.selectedSemiProductUniqueId = selectedMaterialUniqueId
                    toAddPiece.add(newPiece)
                }
            }
            pieceIndex += 1
        }

        if (toAddPiece.isNotEmpty()) {
            context.Group<Piece>().addAll(toAddPiece)
        }

        PlanManager.getInstance(context).splitPiecesToPieceSteps()
    }

    return (totalFulfillRatio, toAdd)
}

/*
 * 国际订单成品未指定批次数：
 * 按库存A -> 库存B -> 生产A -> 生产B逐批分配；
 * 库存分完仍不足时允许继续使用生产。
 */
if (guojiBatchCountLimit <= 0 && remainQty > Constant.EPS) {
    sourcePiece = context.Group<Piece>().find { it.pieceId == firstDemand.pieceId }
    if (sourcePiece == null || sourcePiece.owner == null) return tuple

    var workOrder = sourcePiece.owner
    var batchSupplyList = new List<MaterialSupply>()
    var batchDemandList = new List<MaterialDemand>()
    var batchQtyList = new List<Double>()

    var previewSupplies = new List<MaterialSupply>()
    previewSupplies.addAll(sortedSupplies)
    var previewRemainQty = remainQty.roundHalfUp(6)

    while (previewSupplies.isNotEmpty() && previewRemainQty > Constant.EPS) {
        var supply = previewSupplies.firstOrNull()
        if (supply == null) break
        previewSupplies.remove(supply)

        var currentDemand = demandMap.containsKey(Utils.anyToString(supply.productId))
            ? demandMap.get(Utils.anyToString(supply.productId))
            : null
        if (currentDemand == null) continue

        var availableQty = MaterialSupply.getGuojiBanchengpingAvailableQty(supply).roundHalfUp(6)
        if (availableQty <= Constant.EPS) continue

        var batchQty = minOf(previewRemainQty, availableQty).roundHalfUp(6)
        if (batchQty <= Constant.EPS) continue

        batchSupplyList.add(supply)
        batchDemandList.add(currentDemand)
        batchQtyList.add(batchQty)

        previewRemainQty = (previewRemainQty - batchQty).roundHalfUp(6)
        if (previewRemainQty < 0.0) previewRemainQty = 0.0
    }

    if (previewRemainQty > Constant.EPS) return tuple

    var remainBatchQty = remainQty.roundHalfUp(6)
    var batchIndex = 0

    batchSupplyList.forEach { supply ->
        var currentDemand = batchDemandList[batchIndex]
        var batchQty = batchQtyList[batchIndex]

        remainBatchQty = (remainBatchQty - batchQty).roundHalfUp(6)
        if (remainBatchQty < 0.0) remainBatchQty = 0.0

        var currentDemandQuantity = currentDemand.demandQuantity
        if (currentDemandQuantity <= Constant.EPS) currentDemandQuantity = demandQuantity
        var fulfillRatio = currentDemandQuantity > Constant.EPS
            ? batchQty / currentDemandQuantity
            : 0.0
        totalFulfillRatio += fulfillRatio

        if (!tempFlag) {
            var fulfill = MaterialFulfill.createFulfillByDemandAndSupply(
                currentDemand,
                supply,
                batchQty
            )
            fulfill.ppProjectId = context.projectCode()
            fulfill.ppSceneId = context.sceneCode()
            fulfill.frozenFlag = frozenFlag
            fulfill.isSemiProductInPlant = bomBanchengpingIds.contain(
                Utils.anyToString(currentDemand.materialId)
            )
            fulfill.unFulfillQuantity = remainBatchQty
            toAdd.add(fulfill)

            currentDemand.fulfillQuantity += batchQty
            MaterialSupply.deductGuojiBanchengpingAvailableQty(supply, batchQty)

            if (firstDemand.isFirstConsumeSet) {
                firstDemand.leftRatio -= fulfillRatio
                if (firstDemand.leftRatio < Constant.EPS) firstDemand.leftRatio = 0.0
            }
        }

        batchIndex += 1
    }

    /*
     * 根据实际供应批次拆分 Piece，并保存每个 Piece 对应的 supply uniqueId。
     */
    if (!tempFlag && batchQtyList.isNotEmpty()) {
        var maxSystemNum = workOrder.pieces.maxByOrNull {
            it.systemNum ?? 0
        }?.systemNum ?? 0

        var pieceIndex = 0
        batchQtyList.forEach { batchQty ->
            var pieceQty = batchQty.roundHalfUp(6)
            var selectedMaterialUniqueId = Utils.anyToString(
                batchSupplyList[pieceIndex].uniqueId
            )

            if (pieceQty > Constant.EPS && !selectedMaterialUniqueId.isEmpty()) {
                if (pieceIndex == 0) {
                    sourcePiece.quantity = pieceQty
                    sourcePiece.selectedSemiProductUniqueId = selectedMaterialUniqueId
                } else {
                    maxSystemNum += 1
                    var newPiece = workOrder.createNewPiece(pieceQty, maxSystemNum)
                    newPiece.selectedSemiProductUniqueId = selectedMaterialUniqueId
                    toAddPiece.add(newPiece)
                }
            }
            pieceIndex += 1
        }

        if (toAddPiece.isNotEmpty()) {
            context.Group<Piece>().addAll(toAddPiece)
        }

        PlanManager.getInstance(context).splitPiecesToPieceSteps()
    }

    return (totalFulfillRatio, toAdd)
}

return (totalFulfillRatio, toAdd)
```

## 3. 核料流程与规则梳理

### 3.1 核料目标与处理边界

核料的目标是：基于工单工序的物料投入需求，汇总当前场景可使用的库存、待检库存、采购、跨场景计划和本车间产出等供应，按照替代、配套、批次、库位、手册、有效期和来源等约束进行分配，最终得到：

- 每条需求由哪些供应满足，以及满足数量、未满足数量和供应可用时间；
- 每个批次工序的物料最早可用时间和齐套结果；
- 可发送到 DM 的物料分配明细；
- 可从 DM 回读并保存的核料满足结果。

核料只负责物料可用性与分配，不直接改变工单工艺路线，也不替代排程算法。核料结果通过工序最早可用时间和齐套量影响后续排程。

### 3.2 完整执行流程

| 阶段 | 入口方法 | 处理内容 | 主要输出 |
|---|---|---|---|
| 0. 开关判断 | `enabledCheckInputMaterial` | 读取场景参数 `checkInputMaterial`；为空时按 `false` | 决定是否继续核料 |
| 1. 建立工序投入/产出 | `synInputAndOutputOnStep` | 按场景模式选择工序 BOM 或普通 BOM，建立 `InputOnStep`、`OutputOnStep` | 工序级物料投入和产出 |
| 2. 同步供应 | `synMaterialSupply` | 清空旧供应，重新汇总库存、待检、原料药、采购、跨场景计划和本场景产出 | `MaterialSupply` |
| 3. 扣除外部占用 | `synMaterialSupplyAvailableQuantity` | 读取其他项目/场景的核料分配，按供应唯一 ID 扣除占用 | 当前场景可用数量、剩余数量 |
| 4. 生成需求 | `synMaterialDemand` | 由可核料工序的 `InputOnStep` 生成需求，计算损耗后的需求量 | `MaterialDemand` |
| 5. 清理与恢复冻结分配 | `clearUnfrozenAndReOccupyFrozenMaterialFulfills` | 删除未冻结分配；验证并重新占用有效冻结分配 | 冻结分配、需求满足量、供应剩余量 |
| 6. 重新分配 | `assignSuppliesToMaterialDemands` | 按替代优先级和需求顺序分配供应 | 新增 `MaterialFulfill` |
| 7. 回写工序 | `updateFulfillTimeToPieceStepTime` | 根据分配结果计算工序齐套量和物料最早可用时间 | `PieceStep` 核料结果 |
| 8. 对外同步 | `sendFulfillsToDM` | 发送 PP 分配结果到 DM，再回读 DM 核料满足数据 | DM 数据及本地 `DMFulfill` |

顶层方法 `checkInputMaterial` 为上述阶段记录运行耗时。任一阶段需要独立调用时，也会再次检查核料开关。

### 3.3 前置条件与限制条件

#### 3.3.1 场景开关

- `owner.sceneParameter.checkInputMaterial == true` 才执行核料。
- 参数为空时按 `false` 处理。
- 关闭开关后，不生成新的投入、供应、需求或分配实例。

#### 3.3.2 工序与需求有效性

- 只有 `cannotCheckInputReason == ""` 的 `PieceStep` 才生成物料需求。
- 工序不存在、BOM 不存在、BOM 无效或找不到目标步骤时，不创建对应投入。
- 需求初始量仅由第一优先级投入产生。
- 需求量考虑损耗/收率：`demandQuantity = initDemandQuantity / scrapRate`；损耗率为空或不大于 0 时按 1 处理。
- 数量判断普遍使用 `Constant.EPS`，小于等于阈值的供应或需求视为无效。

#### 3.3.3 BOM 模式

- `orderAndProcessPattern == ModelConstants.Pattern23` 时使用工序 BOM。
- 其他模式使用普通 BOM。
- 工序 BOM 会递归展开下层 BOM；只有没有下层 BOM 的叶子物料才创建工序投入。
- 普通 BOM 中，有 KT 特殊工序约束的物料在指定步骤创建投入；无特殊约束的普通物料在首步骤创建投入。
- 每个订单选中路径的末端步骤创建产出。
- 非半成品订单配置了 `inputProductId` 时，只允许清单中的原料进入核料；半成品订单或清单为空时不限制。

#### 3.3.4 供应有效性

供应来源包括：

1. 可用库存；
2. 待检库存；
3. 分批到货原料药；
4. 采购订单；
5. 采购申请/采购计划；
6. 其他场景生产计划；
7. 本车间工序产出。

共同规则：

- 供应数量必须大于 `Constant.EPS`；
- 待检、采购类供应的释放时间需要叠加检验周期；
- 半成品库存可用日期优先使用 `semiStorageDate`；
- 半成品同时维护实际数量和虚拟数量；
- 所有供应统一生成 `supplySortKey`，作为后续排序依据；
- 其他车间已占用数量会从供应量中扣除，当前场景剩余量保留 7 位精度。

### 3.4 分配规则

#### 3.4.1 需求排序和替代优先级

- 需求按交期、Piece 系统序号和步骤系统序号排序。
- 首先处理每个替代组的第一优先级需求。
- 第一优先级不能完全满足时，才继续处理后续替代优先级。
- `leftRatio` 表示第一优先级需求尚未满足的比例；分配后按实际满足比例扣减。
- 配套组和整组替代要求组内物料共同满足，不能只看单条物料需求。

#### 3.4.2 供应候选过滤

候选供应需要同时通过适用的约束：

- 物料代码或替代物料匹配；
- 供应剩余数量大于阈值；
- 供应可用时间、需求时间和有效期约束；
- 禁用库位约束；
- 供应商约束；
- 上游车间约束；
- 强制换料、不可混用和整组使用约束；
- 国际订单加工贸易手册的物料及库位约束；
- 国际成品使用 BOM 半成品时的指定批次、生产月份和来源约束；
- 采购订单、采购申请、库存、生产计划等来源优先级；
- 半成品实际数量和虚拟数量分别扣减。

#### 3.4.3 供应排序

- 普通供应主要按 `supplySortKey` 排序，核心含义是按可用时间和来源顺序优先使用。
- 国际成品 BOM 半成品规则：
  1. 指定库存批号优先；
  2. 其他库存其次；
  3. 生产计划半成品按供应月份先进先出；
  4. 同月生产计划按可用批量较小者优先；
  5. 其他来源最后兜底。
- 国际批次数需求按供应可用量与目标批量的差值排序，差值越小越优先。

#### 3.4.4 冻结分配

重跑核料时：

- 未冻结的 `MaterialFulfill` 全部删除并重新计算；
- 冻结分配不会直接删除，而是重新验证需求和供应；
- 需求不存在、供应不存在、库存命中禁用库位或不符合加工贸易手册时，冻结分配判为无效；
- 有效冻结分配会重新累加需求满足量、扣减供应剩余量并更新供应可用时间；
- 无效冻结分配从数据库删除，防止继续占用供应。

#### 3.4.5 分配结果

每次有效分配会生成 `MaterialFulfill`，记录：

- 需求和供应唯一 ID；
- 分配数量和未满足数量；
- 供应来源、批号、车间、工厂、释放日期和释放时间；
- 需求产品、物料、工序、订单、交期、替代组和优先级；
- 是否冻结、是否半成品等标识。

国际半成品按实际供应批次分配时，还可能拆分 Piece，并把供应唯一 ID 写入对应 Piece。

### 3.5 齐套与时间回写规则

- `updateFulfillTimeToPieceStepTime` 将分配结果交给 `synAssignResultToPieceSteps`。
- 工序物料最早可用时间取满足该工序需求的供应释放时间。
- 第一优先级或配套组未完全满足时，工序保留未齐套状态。
- 供应分批满足时，需要同时计算已齐套数量和剩余未满足数量。
- 回写前保存原始 `pieceSteps` 集合，避免分配过程中新增或拆分 Piece 导致遗漏。
- 核料结果是排程的输入限制：物料不可用时，工序不能早于物料最早可用时间执行。

### 3.6 DM 同步规则

`sendFulfillsToDM` 包含两个方向：

1. PP → DM  
   - 查询全部 `MaterialFulfill`；
   - 携带 `PPSceneId`、`PPProjectId` 和分配数据；
   - 调用 `DataReceiverManager.synMaterialFulfillFromPP`。

2. DM → PP  
   - 调用 `DataSenderManager.sendIPP_MaterialFulfillToPP`；
   - 删除本地旧 `DMFulfill`；
   - 把返回数据转换为 `DMFulfill`；
   - `uniqueId` 为空的数据不保存；
   - 批量插入或更新有效结果。
