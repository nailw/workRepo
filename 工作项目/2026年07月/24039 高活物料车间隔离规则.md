# 核料禁用库位功能修改说明

**开发建议**：先完成“必改部分”，验证新逻辑能够正确过滤禁用库位；冻结分配属于第二阶段处理，避免一次改动过大。

## 二、增加模型字段

### 1. KTProductRouting

新增字段：

- **字段名**：`NoLocationId`
- **类型**：`String`
- **描述**：禁用库存代码
- **默认值**：`""`

**字段名必须严格一致**（区分大小写）：

```java
routing.NoLocationId
```

**支持格式**：
- 单个库位：`3411`
- 多个库位：`3411,3413`

维护时请勿输入空格，使用英文逗号分隔。

### 2. MaterialSupply

新增字段：

- **字段名**：`sapStockingPointId`
- **类型**：`String`
- **描述**：ERP库存代码
- **默认值**：`""`

> 说明：`InventoryIn` 已有该字段，但转换到 `MaterialSupply` 时未复制，导致核料阶段无法知道库存来自哪个库位。  
> 字段增加后，需按平台流程执行**模型发布**或**数据库同步**。

## 四、新增查询产品禁用库位的方法

**建议**在 `MaterialSupply` 类中新增静态方法：

```java
方法名：queryNoLocationIdsByProductId
所属类：MaterialSupply
返回值：List<String>
```

**参数**：

| 参数名     | 类型     |
|------------|----------|
| context    | Context  |
| productId  | String   |

**方法体**：

```java
var result = new List<String>()
if (productId == "") {
    return result
}
context.KT<KTProductRouting>().filter{ it.productId == productId }.forEach{ routing ->
    var locationIds = Utils.splitStringToStringList(routing.NoLocationId ?? "")
    locationIds.forEach{ locationId ->
        if (locationId != "" && !result.contain(locationId)) {
            result.add(locationId)
        }
    }
}
return result
```

**功能**：
- 支持多个库位
- 自动去重
- 无配置时返回空列表
- 同一产品多条路径时合并禁用库位

## 五、修改公共供应查询

**修改位置**：`MaterialSupply.queryAvailableFromInputSupplies`

```java
var supplyMap = inputSupplies.groupBy { it.productId }
var outputSupplies = supplyMap.getOrDefault(materialDemand.materialId, new List<MaterialSupply>())

// 根据当前生产的成品代码查询禁用库存代码
var noLocationIds = MaterialSupply.queryNoLocationIdsByProductId(conetxt, materialDemand.productId)
// 只过滤ERP库存供应，采购和生产计划不受影响
if (!noLocationIds.isEmpty()) {
    outputSupplies.removeIf{
        MaterialSupply.isInventorySupply(it) &&
        (it.sapStockingPointId ?? "") != "" &&
        noLocationIds.contain(it.sapStockingPointId)
    }
}
```

**注意**：继续使用原有参数名 `conetxt`（即使拼写不标准）。

## 六、关键说明

- **必须使用** `materialDemand.productId`（当前成品代码），而非 `materialDemand.materialId`（BOM物料代码）。
- **只过滤库存供应**：必须保留 `MaterialSupply.isInventorySupply(it)` 判断。
